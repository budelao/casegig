# Requisitos

## Funcionais

- Criar ordem de aplicação
- Criar ordem de resgate
- Consultar ordens
- Consultar posição do cliente
- Validar saldo antes da aplicação
- Validar posição antes do resgate
- Aplicar regra de cut-off por fundo
- Permitir agendamento automático de ordens fora do horário
- Processar ordens agendadas

## Não Funcionais

- Garantir consistência transacional (ACID)
- Suportar concorrência
- Garantir integridade de dados financeiros
- Código organizado e de fácil manutenção
- Facilidade de evolução da arquitetura
